Scheduler

Daily cron · when and how this runs

Daily cron at 02:00 UTC. One make run per day, labeled incremental_run in performance.jsonl. Default schedule installs via make schedule.

Trigger
cron · 0 2 * * * UTC
First time
make init — one-time setup (DDL + dim_date seed + first run)
Daily
make run — ingest → transform → analyze
Backfill
make backfill START=… END=… for historical windows
Reset
make clean && make init — nuke the warehouse + fresh start
Recovery

Failure modes & how they're contained

Four failure modes, each with its own recovery path — plus self-healing for a missed cron. Nothing fails silently.

HTTP
Retries on rate-limit (429), server error (5xx), and timeouts — up to 3 attempts per call, waits 1 s → 30 s between tries, honours Retry-After from the server.
Crashes
orphan running heartbeat in meta.pipeline_runs, surfaced by make doctor
Data
4 silver DQ assertions — unique PK, not-null, bounds, freshness — fail the run on severity='error'
Atomic TX
bronze tmp→rename · silver BEGIN/COMMIT with rollback · dbt tests post-build
Missed run
next make run starts at max(silver.date) + 1 and self-heals
